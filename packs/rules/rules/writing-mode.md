# Writing Mode Rules

Active when creating blog posts, articles, or long-form content.

## Voice
- Match the user's writing voice (see `data/voice-profile.md` if it exists)
- If no voice profile exists, write clearly and directly — not like AI
- Avoid: "delve", "landscape", "synergy", "leverage", "tapestry", "realm", "foster"
- Shorter sentences beat longer ones
- Active voice beats passive voice

## Quality
- Every sentence must earn its place — cut anything that doesn't add meaning
- Lead with the interesting part, not the setup
- Specific beats vague: "revenue grew 40%" beats "revenue grew significantly"
- If you can say it in fewer words, do

## Structure
- Hook in the first two sentences — or lose the reader
- One idea per paragraph
- Transitions should be invisible — if the reader notices them, they're too heavy
